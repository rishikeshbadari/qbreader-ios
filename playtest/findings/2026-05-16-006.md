# 2026-05-16-006 — Same-name non-host could resume during host settings pause

**Flow**: multiplayer, three real simulator clients

**Repro**:
1. Start a multiplayer game with all players using the default `Player` name.
2. Have the host open game settings mid-question.
3. Inspect a non-host client while the settings modal is open on the host.

**Observed**: Non-host clients could see a `Resume game` overlay because pause ownership was compared by display name. With duplicate player names, the app treated the host pause as the local player's own pause.

**Expected**: Non-host clients should see a passive `Player is changing game settings...` overlay and should not be able to resume another player's settings pause.

**Suspected file**: `app/multiplayer/game.tsx`, `context/MultiplayerContext.tsx`, `types/multiplayer.ts`

**Status**: fixed
