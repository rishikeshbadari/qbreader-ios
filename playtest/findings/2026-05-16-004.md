# 2026-05-16-004 — Multiplayer lobby state divergence

**Flow**: mp-host, three physical simulator clients over Supabase Realtime.

**Repro**:
1. Start Expo on port 8084 with `EXPO_PUBLIC_QBREADER_MOCK=1`.
2. Open the app on iPhone 17 Pro, iPhone Air, and iPad (A16).
3. Host a multiplayer game on iPhone 17 Pro.
4. Join the room from iPhone Air and iPad.

**Observed**:
- iPhone Air saw `3/10 players`.
- iPhone 17 Pro and iPad saw `2/10 players`.
- The original host screen lost host controls and showed the non-host ready toggle.

**Expected**: All three clients should agree on the same lobby player list, and the original host should retain host controls.

**Suspected file**: `context/MultiplayerContext.tsx`

**Status**: fixed

**Fix**: Only clients with authoritative room state can send `state:sync`; the host remains authoritative during lobby setup, and malformed sync payloads without `hostId` or `settings` are ignored.
